Module berry

This module is what you use to interface with Berry itself. It facilitates window management, input, rendering, and audio.

Functions

define bind_step(function: Function())

Tells Berry to call a specific function each frame.

You do not control the main application loop yourself when using Berry. Instead, you give Berry a function to call each frame, and it will do so at the correct time, while handling all the other necessary boilerplate (input handling, rendering, etc).

You can bind a new step function at any time, if you so desire. Perhaps that could be useful for something like a title screen, that is sufficiently separate from the rest of the game and could use its own mainloop. For most games, though, simply binding a single step function in game.lily will suffice.

Vars

var save_path: String

A directory appropriate for writing save data in.

You don't have to use this directory, necessarily. However, if all you're doing is writing typical save files, and not interacting with the broader filesystem, this directory is a good idea to use. It will be set to a standard location for application data on the current platform.

Examples
  • On Windows: %appdata%\<game title>\
  • On Mac: ~/Library/Application Support/<game title>/
  • On Linux: ~/.local/share/<game title>/
