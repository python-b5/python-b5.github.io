Class Sound

Sound data that can later be played.

Sound data is loaded from disk - the constructor is passed a path to an audio file to load. Each audio file should only be loaded once, since it would be a waste of memory to load the same sound multiple times. The same Sound object can be reused for as many purposes as necessary.

Both WAV and Ogg Vorbis audio files are supported. Generally, it is best to use WAV for sound effects, and Ogg for music.

constructor(path: String): Sound

public define length: Double

Returns the length of self, in seconds.

public define play(looping: *Boolean, position: *Double, volume: *Double, speed: *Double): PlayingSound

Starts playback of self.

Several options can be configured through optional keyword arguments. They are passed directly into PlayingSound's constructor, and so are roughly equivalent to setting the values manually through PlayingSound's various methods. They are as follows:

  • looping determines whether the PlayingSound should loop upon finishing.
  • position is used as the initial playback position for the PlayingSound, in seconds. It must be between 0.0 and the length of self.
  • volume is used as the initial volume for the PlayingSound, as a multiplier applied to self's original volume.
  • speed is used as the initial speed for the PlayingSound, as a multiplier applied to self's original speed. It must be between 0.0 and 100.0 (the latter limit is inherited from SDL, which Berry uses internally - if a different library were used for sound playback instead, that limit would likely not be in place).
Errors
  • ValueError if any invalid values are passed as options.
